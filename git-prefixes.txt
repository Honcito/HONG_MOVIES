feat: Se utiliza para un nuevo rasgo o funcionalidad en el código. Es uno de los más importantes porque indica que el software ha crecido en capacidades.

    Ejemplo: feat: añade la funcionalidad de búsqueda de películas por género.

fix: Se usa cuando se ha corregido un error en el código. Esto es crucial para el control de calidad.

    Ejemplo: fix: corrige el error que no permitía el login en dispositivos móviles.

docs: Para cambios que solo afectan a la documentación. Esto puede ser desde un archivo README.md hasta comentarios en el código.

    Ejemplo: docs: actualiza el README con instrucciones de instalación.

style: Se refiere a cambios de formato que no modifican la lógica del código. Esto incluye la adición de espacios en blanco, punto y coma, sangría, etc.

    Ejemplo: style: formatea el código para seguir el estilo de la empresa.

refactor: Para un cambio en el código que ni añade funcionalidad ni corrige un error. Normalmente, es una mejora en la estructura o el diseño del código.

    Ejemplo: refactor: optimiza la forma en que se obtienen los datos de la API.

test: Para añadir o corregir pruebas unitarias o de integración.

    Ejemplo: test: añade pruebas para el componente Card.

chore: Para cambios de mantenimiento general que no están relacionados con el código de producción o las pruebas. Esto puede incluir actualizaciones de dependencias, cambios en la configuración de la construcción, etc.

    Ejemplo: chore: actualiza la versión de Vite a la última estable.

build: Se utiliza para cambios que afectan al sistema de construcción o a las dependencias externas.

    Ejemplo: build: actualiza la versión de Node.js en el Dockerfile.

ci: Para cambios en los archivos y scripts de integración continua (CI).

    Ejemplo: ci: añade un paso para ejecutar las pruebas en GitHub Actions.
